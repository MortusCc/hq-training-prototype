# HQ技术培训管理系统（高保真原型）

本项目为《需求分析与系统设计》综合实践的“2.2（2）高保真度原型系统”。原型系统可运行、可交互，用于演示用户可见的系统功能界面与关键业务流程，不依赖后端服务与真实数据库。

## 在线访问（推荐）

无需下载代码、无需本地运行，直接打开以下网址即可在线查看原型：

- https://hq-training-prototype.vercel.app/

## 项目定位

- 目标系统：HQ技术培训管理系统
- 原型范围：培训申请、课程、讲师、学员、报名、收费、签到、调查、统计报表等核心流程的可视化与交互演示
- 数据存储：浏览器 localStorage（演示数据可重置）

## 技术栈

- React + TypeScript
- Vite
- react-router-dom（前端路由）
- localStorage（本地 Mock 数据与会话）

## 功能与页面（按角色）

登录页支持一键进入不同端，快速切换演示角色。

- 经理端
  - 培训申请评审：立项/拒绝，指定执行人，立项后自动生成培训计划（课程草稿）
  - 培训课程总览：查看课程状态与来源（委托培训/公开课）
- 执行人端
  - 课程管理：编辑课程信息、选择讲师、生成培训通知（HTML）、发送开课前提醒、结束课程归档
  - 查看课程评价：查看学员个人评价与工作人员端填写的总体课程评价
  - 讲师资料管理：讲师信息电子化展示
  - 报名与通知：报名名单、确认报名、标记收款
  - 网站通知与导出：导出培训通知 HTML、邮件收件人 CSV、报名名单 CSV
- 学员端
  - 课程浏览/报名：浏览已发布课程、在线填写报名表并提交（区分委托方免收培训费）
  - 我的报名：查看状态，支持模拟缴费与签到（按当前日期演示）
  - 课程评价：对已完成课程提交满意度与建议（再次提交将覆盖）
- 工作人员端
  - 签到名单/资料发放：核对签到名单，为已签到学员发放资料
  - 课程评价/报告：查看学员个人课程评价，填写并保存总体课程评价（用于归档与复盘）
- 统计报表
  - 课程门次、报名人次、培训收入、执行人工作量概览

## 演示建议（录屏推荐顺序）

1. 经理端：评审培训申请 → 立项并生成课程草稿（分配执行人）
2. 执行人端：编辑课程 → 生成培训通知（HTML）并发布 → 导出 HTML/CSV → 查看报名名单并发送提醒
3. 学员端：浏览课程 → 在线填写报名表并提交 → 在“我的报名”中模拟缴费与签到 → 课程结束后在“课程评价”提交评价
4. 工作人员端：核对签到名单并发放资料 → 查看学员个人评价 → 填写总体课程评价并保存
5. 执行人端：在“课程管理”中查看课程评价（个人评价 + 总体评价）
6. 统计报表：展示课程、报名、收入与执行人工作量统计

提示：页面右上角支持修改“当前日期”，用于模拟开课前提醒、签到、结课、评价等时间点。

## 运行方式

若仅用于查看与演示原型，优先使用上面的“在线访问”链接。

在本目录下执行：

```bash
npm install
npm run dev
```

浏览器打开（默认）：

- http://127.0.0.1:5173/

构建与预览：

```bash
npm run build
npm run preview
```

## 数据说明

- 初始演示数据来自 `src/lib/seed.ts`
- 数据与会话保存在浏览器 localStorage
- 可在页面右上角点击“重置演示数据”恢复初始状态

## 目录结构（核心）

- `src/pages/`：各端页面（manager / executor / student / onsite）
- `src/state/`：演示数据与会话管理（localStorage + React Context）
- `src/lib/`：种子数据、导出文件、日期工具
- `dist/`：构建输出（由 `npm run build` 生成）

## 导出说明（对接外部系统的“接口”模拟）

- 培训通知：导出为 HTML 文件（用于提供给宣传网站发布）
- 邮件收件人/报名名单：导出为 CSV 文件（可作为 Excel 导入邮件系统/名单管理的输入）
